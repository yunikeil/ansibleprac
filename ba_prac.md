# Методология сбора информации и анализ требований для внедрения CRM-системы

Данный документ описывает комплекс мероприятий по исследованию бизнес-процессов организации с целью формирования функциональных требований к CRM-системе.

## 1. Способы сбора информации

Для обеспечения полноты и достоверности данных планируется использование следующих методов:

| Метод | Обоснование | Ожидаемый результат |
| :--- | :--- | :--- |
| **Интервьюирование** | Позволяет выявить специфику работы на каждом участке, определить индивидуальные должностные обязанности и субъективные трудности сотрудников. | Реестр пользовательских требований, описание текущих рабочих процессов «как есть» (AS-IS). |
| **Метод прямого наблюдения (Job Shadowing)** | Исключает искажение информации, возникающее при устном описании задач. Позволяет зафиксировать реальные временные затраты и скрытые потери. | Карта фактического процесса, выявленные узкие места (бутылочные горлышки) и точки дублирования данных. |
| **Анализ документации и артефактов** | Изучение текущих инструментов учета (Excel-таблицы, журналы) необходимо для определения структуры базы данных. | Спецификация полей, сущностей и связей, необходимых для миграции данных в новую систему. |
| **Проведение рабочих встреч (Workshops)** | Совместное обсуждение минимизирует конфликты интересов между подразделениями и помогает выстроить целевую модель процесса. | Согласованный перечень приоритетных требований и концепция работы в CRM (TO-BE). |

---

## 2. Заинтересованные лица

В процессе сбора информации задействованы следующие группы участников:

1.  **Администратор:** Основной оператор системы. Предоставляет данные о цикле обработки входящего обращения, ведении клиентской базы и текущих проблемах оперативного учета.
2.  **Менеджер по закупкам:** Эксперт по складской логистике. Формирует требования к модулю управления запасами, интеграции заказов с поставками и контролю остатков.
3.  **Флористы:** Исполнители производственного этапа. Определяют требования к визуализации задач, передаче спецификаций заказов и фиксации этапов готовности продукции.
4.  **Курьер:** Ответственный за логистику последней мили. Передает требования по мобильному доступу к адресам, статусам доставки и механизмам подтверждения получения заказа.
5.  **Бухгалтер (аутсорсинг):** Контролер финансовых потоков. Определяет требования к финансовой отчетности, выгрузке первичных данных и интеграции с учетными системами (например, 1С).
6.  **Руководство компании:** Инициатор проекта. Определяет стратегические цели, ключевые показатели эффективности (KPI) и утверждает приоритеты автоматизации.

## 3. Список вопросов

### Администратор
*   Опишите пошагово процесс обработки заказа от момента первичного обращения до передачи в работу.
*   Какие категории данных являются обязательными при регистрации нового контрагента в текущих таблицах Excel?
*   Каким образом осуществляется информирование клиента о статусе выполнения заказа на данный момент?
*   Какие операции в текущем формате работы вызывают наибольшие временные затраты или приводят к ошибкам?
*   Какие функции автоматического уведомления клиентов (SMS, Email) критически необходимы для вашей работы?

### Менеджер по закупкам
*   Каким образом происходит консолидация данных о потребности в материалах на основе поступивших заказов?
*   Опишите текущий алгоритм мониторинга складских остатков и взаимодействия с поставщиками.
*   Какая информация о заказе необходима вам для своевременного обеспечения производства материалами?
*   Требуется ли внедрение системы автоматических уведомлений при достижении критического минимума запасов?

### Флористы
*   В каком виде вы получаете техническое задание на сборку композиции (устно, мессенджеры, бумажные носители)?
*   Достаточно ли информации о предпочтениях клиента содержится в текущих заявках для качественного выполнения работы?
*   Каким образом фиксируется факт начала и завершения работы над конкретным заказом?
*   Требуется ли вам доступ к календарю заказов для планирования нагрузки на рабочую смену?

### Курьер
*   Каким способом осуществляется передача маршрутных листов и контактных данных получателей?
*   Опишите порядок действий в случае возникновения нештатных ситуаций при доставке (отсутствие клиента, изменение времени).
*   Требуется ли функционал формирования автоматической отчетности о завершении доставки через мобильный интерфейс?
*   Необходима ли автоматизация отправки сервисных сообщений клиенту при выезде курьера на маршрут?

### Бухгалтер (аутсорсинг)
*   Какие типы финансовых отчетов (выручка, затраты, рентабельность) необходимы для ведения учета?
*   Каков регламент и периодичность передачи данных о продажах из оперативного учета в бухгалтерию?
*   Существует ли потребность в автоматической генерации счетов и актов выполненных работ непосредственно из CRM?
*   Необходим ли контроль дебиторской задолженности и автоматизация напоминаний об оплате?

### Руководство компании
*   Какие ключевые бизнес-проблемы должны быть решены в первую очередь после внедрения системы?
*   Какие метрики эффективности работы персонала (KPI) должны быть доступны в виде аналитических отчетов?
*   Каковы долгосрочные цели использования клиентской базы (маркетинговые рассылки, программы лояльности, анализ LTV)?
*   Какие процессы подлежат автоматизации в приоритетном порядке в рамках выделенного бюджета?
